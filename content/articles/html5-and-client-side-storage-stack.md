I&rsquo;m resurrecting an old project of mine now that technology has caught up.
Even so, it&rsquo;s amazing how much time I&rsquo;ve had to spend just trying to find the right tools to use to fulfill my requirements.

## Requirements

- **Cross-Browser**: Needs to work in every modern browser, including mobile.
- **Persistant offline storage**:
In order to support all browsers I must also support the following technologies:
  - IndexedDB
  - WebSQL
  - localStorage
  - Some fallback if none of the above are available..
- **Gracefully degrade** functionality on less capable browsers.
- Encourage users to **modernize** their web experience.
- **Optimize** layout according to the viewable space in the browser.
- **Sane**, consistant starting point for styling pages.
- A **clean**, extensible way of interacting with Javascript.

## Client-Side Tools

- <a href="http://www.initializr.com/" target="_blank">Initializr</a> - Provides a nice frontend scaffold:
  - <a href="http://www.modernizr.com/" target="_blank">Modernizr</a>: Detect browser capabilities.
  - <a href="http://jquery.com/" target="_blank">Jquery</a>: Improves and tidies interactions with Javascript objects.
  - <a href="https://github.com/scottjehl/Respond#readme" target="_blank">Respond.js</a>: Responsive web, layout optimized to viewable space.
  - <a href="http://html5boilerplate.com/" target="_blank">HTML5Boilerplate</a>: Sane and consistant default stylesheet.
- <a href="http://westcoastlogic.com/lawnchair/" target="_blank">Lawnchair</a>: Provides a cohesive API for client-side offline storage.

## Server-Side Tools

Right now I&rsquo;m using some other tools to create a functional prototype of the user experience:

- <a href="https://github.com/blahed/frank#readme" target="_blank">Frank</a>: A static website generator.
- <a href="http://haml-lang.com/" target="_blank">Haml</a>: Because HTML&rsquo;s syntax sucks.
- <a href="http://sass-lang.com/" target="_blank">Sass</a>: Because CSS&rsquo;s syntax both sucks and is lacking in many areas.
- <del><a href="http://mustache.github.com/" target="_blank">Mustache</a>: Because logic doesn&rsquo;t belong in views.</del> - Frank is having issues loading Mustache templates, even after I spent some time <a href="https://github.com/acook/frank" target="_blank">hacking</a> at it.
- <a href="http://jashkenas.github.com/coffee-script/" target="_blank">Coffeescript</a>: Because Javascript is great, but its syntax sucks.

I&rsquo;m not sold on Frank yet, there was some other static site generators I was considering so if Frank starts getting in my way I&rsquo;ll give them a try.

<p>If you&rsquo;re like <a href="http://twitter.com/anthony_m_cook" target="_blank">me</a>, you use <a href="http://en.wikipedia.org/wiki/Interactive_Ruby_Shell" target="_blank">irb</a> for all kinds of things. From just running quick ruby experiments to figuring out how many <a href="http://rubydoc.info/docs/ruby-core/1.9.2/Time" target="_blank">days</a> (9364) you&rsquo;ve been alive.</p>

<p>I created a bookmark in <a href="http://iterm.sourceforge.net/" target="_blank">iTerm</a> to run <a href="http://ruby-doc.org/docs/ProgrammingRuby/html/irb.html" target="_blank">irb</a> when I hit <a href="http://jrgraphix.net/r/Unicode/2300-23FF" target="_blank">⌘</a>-control-0. This was fine until I realized it was using the system&rsquo;s <a href="http://stackoverflow.com/questions/3474875/why-does-mac-os-x-come-with-ruby-rails" target="_blank">ruby</a> install, and not the <a href="http://rubydoc.info/docs/ruby-core/1.9.2" target="_blank">version</a> I installed with <a href="http://rvm.beginrescueend.com/" target="_blank">RVM</a>.</p>

<p>I eventually realized the problem was the system environment doesn&rsquo;t know about my <a href="http://dotfiles.org/" target="_blank">.bash_profile</a>, and therefore my modified <a href="http://www.linuxheadquarters.com/howto/basic/path.shtml" target="_blank">PATH</a> variable nor the require <a href="http://rvm.beginrescueend.com/rvm/install/" target="_blank">RVM magic</a>.</p>

<p>So, to get all that, this is the command to use to make sure you&rsquo;re getting the same irb you expect from your commandline:</p>

<pre><code>bash --login -c irb
</code></pre>

<p><br/>
Yay it works!<br/>
⦂∀</p>
